# 0621 Implementation Plan

## Role

You are the local implementation agent. Follow this document as the entry point, then execute the ToDo files in `docs/todos/0621/` in order. Do not skip tests. Update `docs/todos/0621-todo.md` checkboxes as each item is completed.

## Goal

Move the current PHP Agent from a proof-of-concept raw command runner toward a bounded XREA worker:

- authenticated HTTP API,
- capability-oriented task schema,
- native PHP filesystem operations with path sandboxing,
- raw shell execution disabled by default and stricter when enabled,
- optional file-backed task status store,
- verified PHAR packaging.

This is implementation work. The architecture analysis is already captured in:

- `docs/feasibility/php_agent_design.md`
- `docs/reconsidered_plan.md`

## Target Request Model

Keep backward-compatible `ping` and `info`. Replace raw HTTP `exec` as the primary path with capability execution.

Recommended request shape:

```json
{
  "action": "task.execute",
  "capability": "fs.read",
  "input": {
    "path": "README.md"
  }
}
```

Recommended response shape should continue using `TaskResult::toJson()`:

```json
{
  "status": "success",
  "data": {},
  "error": null,
  "exit_code": 0,
  "duration": 0.001
}
```

## Global Constraints

- PHP version target is `>=8.1`.
- Do not add a web framework.
- Do not add runtime dependencies unless a ToDo explicitly permits it.
- Keep classes small and namespaced under `Xrea\Agent`.
- Use strict types in new PHP files:

```php
declare(strict_types=1);
```

- Keep public HTTP responses JSON-compatible through `TaskResult`.
- Do not throw uncaught exceptions from public request handling.
- Prefer native PHP filesystem functions over shell commands.
- Treat raw shell execution as a high-risk optional capability.
- Tests must be deterministic and must not depend on the user's home directory.

## Error Handling Rules

Use `TaskResult::error($message, $exitCode = 1, $duration = null)` for expected validation and policy failures.

Suggested error style:

- Missing required field: `Missing required field: <name>`
- Invalid capability: `Invalid capability: <name>`
- Unauthorized HTTP request: JSON `{"status":"error","error":"Unauthorized"}` with HTTP `401`
- Path outside workspace: `Path is outside workspace`
- Disabled capability: `Capability disabled: <name>`
