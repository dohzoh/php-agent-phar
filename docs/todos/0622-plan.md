## Goal

Fix `PathSandbox` workspace escape vulnerability: paths with non-existing intermediate directories and `..` traversal are incorrectly allowed by the prefix-based `isInsideWorkspace()` check because `..` segments are not resolved before comparison.

This is a security hardening fix for the path sandboxing layer described in `docs/reconsidered_plan.md`.

## Current Context

### Bug Analysis

The sandbox currently allows paths like `nonexistent/../../outside/file.txt` with workspace `/tmp/ws`:

1. `$fullPath = /tmp/ws/nonexistent/../../outside/file.txt`
2. `file_exists()` → false (nonexistent/ が存在しないため stat が fail)
3. 祖先探索: `/tmp/ws/nonexistent/../../outside` → `/tmp/ws/nonexistent/../..` → `/tmp/ws` (is_dir=true, exit)
4. `realpath('/tmp/ws')` → `/tmp/ws`
5. `isInsideWorkspace('/tmp/ws', '/tmp/ws')` → true
6. 再構成: `fullPath = /tmp/ws/../../outside/file.txt`
7. `isInsideWorkspace('/tmp/ws', '/tmp/ws/../../outside/file.txt')` → **prefix match → true (BUG!)**
8. `realpath('/tmp/ws/../../outside/file.txt')` → false (nonexistent が中間にないため期待通り解決されるが、`..` により `/tmp/outside/file.txt` を指す)

### Root Cause

`isInsideWorkspace()` は文字列 prefix 比較のみ行う。`..` を解決した後のパスが workspace 内かどうかを確認していない。

### Existing Defenses (secondary)

- `ReadCapability::execute()` 内の `is_file($resolved)` — 中間非存在で stat fail → "Not a file"
- `WriteCapability::execute()` 内の `is_dir($parentDir)` — 中間非存在で stat fail → mkdir 試行
- ただしこれらの二次防御に依存するのは安全ではない
